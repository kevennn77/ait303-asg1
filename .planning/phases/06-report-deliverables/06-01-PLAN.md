    **Write Section 4 — Discussion (~2 pages)**

    **4.1 Model Performance Analysis (~0.75 page)**
    - Why SVM (TFIDF + Lemmatized, F1=0.9091) outperformed BiGRU (SG, F1=0.8994):
      - SVM + TFIDF captures n-gram features explicitly; BiGRU learns representations from word embeddings
      - For 50K balanced samples, handcrafted features (TFIDF + lemmatization) are competitive with deep learning
      - BiGRU training is stochastic (mini-batch SGD); SVM solves convex optimization with global optimum
    - Domain shift caveat: models trained on IMDB movie reviews, applied to Best Buy product reviews
      - Language domain differs (movie critique vs product review vocabulary)
      - This means reported F1 scores reflect IMDB test performance, not Best Buy accuracy
    - Convergence note: CountVectorizer models required max_iter=5000 for full convergence (ill-conditioned features due to sparsity); TfidfVectorizer models converged at default max_iter=2000 (TFIDF normalization mitigates conditioning issues)

    **4.2 Aspect Extraction Quality (~0.75 page)**
    - **LDA** (C_v=0.441): moderate coherence, reasonable topic separation. Limitations: requires manual K selection, topics can be generic
    - **BERTopic** (48 topics, 1,051 outliers): fine-grained but high outlier rate (~33%) suggests HDBSCAN threshold may be too strict. Many topics need manual merging
    - **CorEx** (TC=13.19): anchored semi-supervised approach produces directly actionable aspects aligned with our 8 categories. No topic count tuning needed — anchors guide the model
    - **Verdict**: CorEx's semi-supervised anchoring most practical for targeted aspect analysis, but LDA/BERTopic exploration reveals latent themes the anchors might miss

    **4.3 Challenges (~0.5 page)**
    - **Web scraping**: Akamai bot protection required TLS fingerprint impersonation (curl_cffi) and hidden API discovery. Randomized delays (1-3s) prevented rate limiting
    - **Aspect imbalance**: CorEx's Comfort/Portability aspect dominated due to frequent keyword matches (e.g., "comfortable", "fit") — other aspects like Battery Life had fewer anchor matches
    - **Small denominators**: Some product-aspect pairs had fewer than 3 reviews → excluded from ranking. This disproportionately affects niche features (Battery Life, Connectivity)
    - **Domain adaptation**: IMDB-trained sentiment model applied to Best Buy domain — no domain adaptation techniques used, performance on Best Buy reviews is unmeasured

    **Write Section 5 — Conclusion (~1 page)**

    - **Summary**: Built end-to-end ABSA pipeline: 4 sentiment models → scrape → 3 aspect extractors → rank → recommend. Best sentiment model: SVM + TFIDF + Lemmatized (F1=0.9091). Best product: Altec Lansing Jolt Mini Lifejacket (0.900 composite)
    - **Key Insight**: For this task scale (50K training, 3K inference), traditional ML (SVM + TFIDF) outperformed deep learning (BiGRU) — feature engineering and convex optimization win at this data volume
    - **Limitations**: domain shift between IMDB and Best Buy, aspect imbalance in CorEx, small denominators for rare aspects, single aspect label per review
    - **Future Work**: cross-domain sentiment validation, multi-label aspect assignment, weighted aspect importance based on user preference, BERT-based sentiment models for domain adaptation
