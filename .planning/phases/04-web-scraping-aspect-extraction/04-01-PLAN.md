Create `bestbuy_scraper.py` — a standalone executable Python script with the following structure.

**Shebang and docstring:** `#!/usr/bin/env python3` followed by a docstring: `"""Scrape Best Buy Bluetooth & Wireless Speaker reviews for aspect-based sentiment analysis."""` — following `_create_notebook.py` pattern at lines 1-2.

**Global constants:**
- `USER_AGENTS` — list of 3 user-agent strings from RESEARCH.md lines 278-282 (Windows Chrome, Mac Chrome, Linux Chrome).
- `CATEGORY_URL` — the Best Buy Bluetooth & Wireless Speakers category URL with `?intl=nosplash` appended, per RESEARCH.md Pitfall 1: `"https://www.bestbuy.com/site/portable-audio/portable-speakers-docks-radios/pcmcat310200050004.c?id=pcmcat310200050004&intl=nosplash"`
- `OUTPUT_DIR` = `"data_asg/bestbuy"`
- `PRODUCTS_DIR` = `f"{OUTPUT_DIR}/products"`
- `MIN_REVIEWS` = 80 (per D-23)
- `MAX_PRODUCTS` = 40 (upper bound per D-23)
- `MAX_REVIEW_PAGES` = 20 (safety cap to prevent infinite loops, per RESEARCH.md line 769)
- `REQUEST_TIMEOUT` = 30

**Function: `fetch_page(url)`** — per RESEARCH.md Pattern 1 lines 71-77:
- Generates `delay = random.uniform(1.0, 3.0)` then `time.sleep(delay)`
- Creates headers dict with random `User-Agent` from USER_AGENTS and `Accept-Language: en-US,en;q=0.9`
- Uses `requests.Session()` (session object defined at module level) to `get(url, headers=headers, timeout=REQUEST_TIMEOUT)`
- Calls `resp.raise_for_status()` and returns `resp.text`
- Wrapped in try/except: catches `requests.exceptions.RequestException`, prints error with URL, returns empty string

**Function: `parse_listing(soup)`** — extracts product cards from category page:
- Finds product list container (look for `li` elements with class containing `sku-item` or `product-item`, or `div` with data-product-sku attribute)
- For each product element: extracts `sku` (from `data-sku-id` attribute or from URL pattern), `name` (from heading or alt text), `price` (from price element), `rating` (from star rating element or `data-rating`)
- Appends dict `{"sku": sku, "name": name, "price": price, "rating": rating}` to list
- Handles missing data gracefully: if a field can't be parsed, set to None rather than throwing
- If no products found with primary selector, fall back to `script[type="application/ld+json"]` parsing with JSON regex extraction

**Function: `parse_reviews(soup)`** — extracts individual reviews from review page:
- Finds review containers by searching for `div` with class containing `review-item` or `review-content`
- For each review: extracts `title` (heading), `text` (paragraph with review body), `rating` (from star element or numeric class), `date` (from date/time element or text containing date pattern)
- Appends dict `{"title": title, "text": text, "rating": rating, "date": date}` to list
- Falls back to `script[type="application/ld+json"]` with JSON parsing if structural selectors fail (per RESEARCH.md Pitfall 2)
- Handles empty review sets gracefully — returns empty list, doesn't crash

**Function: `scrape_product_listing(category_url, max_products=MAX_PRODUCTS)`**:
- Calls `fetch_page(category_url)` with first page
- Creates BeautifulSoup object with `"html.parser"`
- Calls `parse_listing(soup)` to extract products
- Checks for "next page" pagination link and continues if more products needed
- Returns list of product dicts, capped at `max_products`

**Function: `scrape_reviews(sku, product_name, max_reviews=100)`**:
- Iterates review pages starting from page 1: `f"https://www.bestbuy.com/site/reviews/{product_name}/{sku}?page={page}"`
- For each page: `fetch_page(url)` → `parse_reviews(soup)`
- If parse_reviews returns empty list, break (no more pages)
- Cap at `max_reviews` reviews per product
- Cap at `MAX_REVIEW_PAGES` (20) iterations as safety limit per RESEARCH.md T-04-03 mitigation

**Function: `save_product_data(product, reviews, output_dir=PRODUCTS_DIR)`**:
- Creates `output_dir` if not exists (`os.makedirs(output_dir, exist_ok=True)`)
- Builds output dict: `{**product, "reviews": reviews, "total_reviews_collected": len(reviews)}`
- Writes to `f"{output_dir}/{product['sku']}.json"` with `json.dump(indent=2)`

**Function: `save_consolidated(products_data, reviews_data, output_dir=OUTPUT_DIR)`**:
- Creates `output_dir` if not exists
- Writes `products.json`: list of all product metadata dicts (without reviews)
- Writes `all_reviews.csv`: CSV with columns `product_name, review_text, rating, date`
  - Includes every review from all products
  - Uses `csv.DictWriter` with the four named columns

**Function: `main()`** — top-level orchestrator:
1. Print start banner
2. Create output directories: `os.makedirs(PRODUCTS_DIR, exist_ok=True)`
3. Call `scrape_product_listing(CATEGORY_URL, MAX_PRODUCTS)` to get product catalog
4. Print summary: "Found {N} products"
5. For each product, call `scrape_reviews(sku, name)` to collect reviews
6. Print progress: "({i}/{total}) {name} — {count} reviews"
7. Call `save_product_data(product, reviews)` for each
8. Collect all reviews across all products
9. Call `save_consolidated(products_data, all_reviews)`
10. Print final summary:
   - Total products scraped
   - Total reviews collected
   - Output file locations
   - Products with fewer than 80 reviews (warning)

**Module-level `if __name__ == '__main__': main()`** guard (per _create_notebook.py line 447 pattern).

**Error handling throughout:**
- Network errors in `fetch_page`: caught and logged, returns empty string
- Parsing errors in any `parse_*`: caught with `except Exception`, logged, returns empty list gracefully
- File write errors: caught and logged, continues with remaining products
- KeyboardInterrupt: caught at main level, prints current progress before exiting

**Anti-detection measures (per D-21):**
- All 3 user agents from RESEARCH.md
- `Accept-Language: en-US,en;q=0.9` header
- `?intl=nosplash` append to prevent international redirect (Pitfall 1)

**Output directory creation:** The script creates `data_asg/bestbuy/` and `data_asg/bestbuy/products/` as needed.
